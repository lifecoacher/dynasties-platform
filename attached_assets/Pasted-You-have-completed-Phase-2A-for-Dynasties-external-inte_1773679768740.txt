2. RECOMMENDATION ENRICHMENT RULES

Update recommendation generation rules so the following types become intelligence-aware:

ROUTE_ADJUSTMENT
- trigger or strengthen when destination/origin congestion is elevated
- trigger or strengthen when lane disruption or weather risk is elevated

DELAY_WARNING
- incorporate congestion, disruptions, weather, and vessel/port call anomalies

COMPLIANCE_ESCALATION
- incorporate sanctions/denied-party linking confidence and severity

CARRIER_SWITCH
- strengthen when lane stress or vessel-related issues suggest a carrier change

RISK_MITIGATION
- raise severity when multiple external signals stack together

PRICING_ALERT
- fully implement this type using lane market signals, congestion pressure, and disruption premiums
- do not leave this as a placeholder

INSURANCE_ADJUSTMENT
- strengthen if weather/disruption risk materially increases exposure

Requirements:
- keep deterministic-first logic
- centralize enrichment thresholds/config
- preserve recommendation fingerprinting/idempotency behavior
- only create materially different recommendations when signal changes justify it
