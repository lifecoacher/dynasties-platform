1. TENANT-SPECIFIC POLICY ENGINE

Create persistent tenant policy models/tables for configurable logic, including:
- recommendation thresholds
- booking gate thresholds
- SLA rules
- auto-task creation rules
- escalation timings
- intelligence weighting
- lane/carrier/entity risk tolerances
- strategic recommendation sensitivity

Requirements:
- safe defaults
- versioning/audit trail for policy changes
- role guards for who can edit policies
- clear separation between global defaults and tenant overrides

2. POLICY SIMULATION / WHAT-IF TUNING

Build a deterministic simulation layer so a user can test policy changes before applying them.

Support scenarios like:
- raising/lowering compliance thresholds
- changing booking hold rules
- changing auto-task creation sensitivity
- adjusting carrier/lane strategy thresholds

Outputs should estimate:
- affected shipments/recommendations/tasks
- changed booking decisions
- changed escalation/task volumes
- changed strategic recommendations

3. CUSTOMER-FACING REPORTING / EXPORTS

Add report generation and export support for:
- executive summary
- portfolio risk
- lane strategy
- carrier allocation
- value attribution
- recommendation performance

Start with:
- API endpoints
- UI report pages/cards
- CSV / JSON export
- structured report objects suitable for future PDF export

4. CONFIGURABLE OPERATING MODES

Create named operating modes such as:
- advisory
- approval_heavy
- semi_autonomous
- high_compliance
- margin_protection
- disruption_sensitive

Each mode should map to policy bundles and be visible/editable in the UI.

5. ROLE-BASED VIEW / PERMISSION REFINEMENT

Refine permissions and UI surfacing for roles such as:
- operator

- Control Tower
