Build a production-grade Compliance Agent inside the existing Dynasties Replit project. This must integrate with the current architecture, existing shipment data model, current AI layer, and UI. Do NOT create a separate prototype or duplicate systems.

WORKING STYLE
- Plan first before coding.
- Inspect the existing repo and reuse all current services.
- Do not introduce parallel systems or duplicate logic.
- Build incrementally with checkpoints.
- Keep everything production-oriented and typed.
- Do not overwrite working code.

PRIMARY GOAL
Implement a real Compliance Agent that evaluates shipment data, detects compliance risks, checks restricted/sanctioned patterns, and returns structured decisions that affect workflow.

---

AI PROVIDER INTEGRATION (CRITICAL)

1. Inspect the codebase for an existing AI provider.
   - If an AI service already exists (Anthropic or otherwise):
     - Reuse it.
     - Do NOT create a second AI client.
     - Route all AI calls through the existing abstraction.

2. If no AI provider exists:
   - Implement Anthropic using ANTHROPIC_API_KEY.
   - Create a wrapper: lib/ai/anthropic.ts
   - Use a clean, reusable interface.

3. Fix existing issues if present:
   - Prevent crashes if API key is missing
   - Use lazy initialization
   - Add error handling and fallbacks

4. LLM usage rules:
   - Input must be structured
   - Output must be structured JSON
   - Low temperature (deterministic)
   - Used for reasoning and explanation only
   - NOT used for hard compliance decisions

5. System must work WITHOUT AI:
   - Rules engine runs first
   - AI enhances reasoning, not replaces it

---

CORE FUNCTION

The Compliance Agent must:

- Accept shipment/trade data
- Evaluate:
  - counterparties
  - origin/destination
  - commodity
  - documentation
- Detect:
  - missing fields
  - suspicious patterns
  - restricted-party risks (via provider abstraction)
- Return structured output:

{
  shipmentId,
  status: CLEAR | REVIEW | BLOCK,
  riskLevel: LOW | MEDIUM | HIGH,
  confidence,
  issues: [],
  recommendedActions: [],
  reasoningSummary,
  sourceChecks: [],
  reviewedAt
}
