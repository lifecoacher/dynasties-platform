
You are a principal full-stack engineer, product architect, and systems designer working on a SaaS billing + receivables platform.

Your objective is to transform the system from a passive billing dashboard into a revenue-generating financing platform with a complete, demo-ready flow.

This must be implemented fully, coherently, and without placeholders.

⸻

CORE OBJECTIVE

Enable this end-to-end loop:

Invoice → Overdue → Financing Offer → Accept Financing → Funds Advanced → Platform Revenue → Reflected in Dashboard

This loop must be:
	•	technically correct
	•	visually clear
	•	instantly understandable
	•	fully consistent across backend + frontend

⸻

SECTION 1 — DATA MODEL EXTENSION

Extend all relevant models (invoice, receivable, etc.) with:
	•	financingStatus: (“NONE” | “ELIGIBLE” | “OFFERED” | “ACCEPTED” | “FINANCED” | “REPAID”)
	•	eligibleForFinancing: boolean
	•	financingOfferedAt: timestamp
	•	financingAcceptedAt: timestamp
	•	financedAmount: number
	•	financingFee: number
	•	platformRevenue: number
	•	providerRate: number
	•	platformSpread: number
	•	customerRate: number

Ensure:
	•	persistence in database
	•	correct typing across backend and frontend
	•	no undefined/null issues

⸻

SECTION 2 — FINANCING RULE ENGINE

Implement a deterministic rule engine:

IF:
	•	invoice.status === “OVERDUE”
	•	outstandingAmount > 0

THEN:
	•	eligibleForFinancing = true
	•	providerRate = 0.025
	•	platformSpread = 0.005
	•	customerRate = 0.03

CALCULATIONS:
	•	financingFee = outstandingAmount * customerRate
	•	financedAmount = outstandingAmount - financingFee
	•	platformRevenue = outstandingAmount * platformSpread

This logic must run:
	•	on invoice fetch
	•	on receivables computation
	•	consistently across API + UI

No duplication. Centralize logic.

⸻

Implement strict lifecycle:

OVERDUE → ELIGIBLE → OFFERED → ACCEPTED → FINANCED → REPAID

Requirements:
