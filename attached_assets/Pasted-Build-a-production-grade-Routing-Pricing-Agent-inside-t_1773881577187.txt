CORE FUNCTION

Given a shipment, the agent must:

1. Analyze:
   - origin / destination
   - ports
   - commodity
   - weight / volume
   - shipment type (air, ocean, etc.)
   - incoterms if present
   - compliance + documentation signals (if available)

2. Generate route options:
   - direct route
   - alternative route(s)
   - optional fallback route

3. Estimate pricing:
   - base cost estimate
   - breakdown (freight, handling, etc. if possible)
   - cost range if uncertainty exists

4. Evaluate:
   - transit time
   - cost vs speed tradeoffs
   - compliance or documentation friction risks

5. Return structured result:

{
  shipmentId,
  recommendedRoute,
  routeOptions: [],
  pricing: {
    estimatedCost,
    currency,
    breakdown,
    confidence
  },
  transitEstimates: [],
  riskFactors: [],
  recommendationSummary,
  reasoning,
  createdAt
}

ROUTING LOGIC

Implement deterministic routing first:

- map origin → destination
- generate:
  - primary route
  - 1–2 alternative routes
- consider:

  - known trade lanes (basic rules)
  - fallback when missing ports

Do NOT over-engineer. Keep it believable.

---

PRICING LOGIC

  - cost range
  - optional breakdown

Pricing must be:
- consistent
- explainable
- not random
